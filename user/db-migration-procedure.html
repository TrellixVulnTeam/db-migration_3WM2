Database Migration ProcedureÂ¶

The following steps should be executed in order.

Note

You must have ssh-agent running in your shell session.

  1. Provision and bootstrap an EC2 instance

    The following azanium sub-command below, when run, will print out the ssh commands needed for the next step.

    AWS_PROFILE="${USER}"
WB_DATA_RELEASE="WS254"
azanium --profile "${USER}" cloud init "${WB_DATA_RELEASE}"

  1. Connect to the EC2 instance using ssh, run db-migration commands

    Note

    The commands below will emit their current progress to the console, and will also print out the location of a log file for more detailed output.

    Use the ssh command printed from step 1 to connect to the EC2 instance.

    Note

    You may need to add the identity to ssh-agent first.

    ssh-add ~/.ssh/wb-db-migrate.pem

    Then, using either screen or tmux session, e.g:

    tmux new-session -s azanium-commands

    issue the following command:

    azanium migrate

  1. Terminate the EC2 instance

    Warning

    The following command will shut down the instance and destroy all data.

    azanium cloud --profile $USER terminate

Should all steps complete successfully, the migration process is now complete.

If you stopped after Step 4 due to data inconsistency, or an error occurred during any of the other steps, please ensure to eventually run Step 5.
